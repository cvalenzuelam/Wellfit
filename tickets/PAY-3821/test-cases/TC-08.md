Validate that App Insights records Treasury API failure telemetry after SFTP transport failure

**AC Linked:** PAY-3821 / AC-1

**AC Description:** Failure is observable in App Insights for role Wellfit Treasury API using correlation id and/or exception traces.

**Preconditions**

- TC-05 **`correlationId`** captured.
- Access to App Insights resource **`stage-insights`**.

**Expected**

- Traces and/or **exceptions** for **Wellfit Treasury API** related to the failure run (e.g. **`SocketException`** / connect timeout).
- Query by **`correlationId`** returns send-stage activity for the triggered run.
- Structured fields **`BatchId`** / **`UnsentBatches`** may or may not appear in STAGE; if absent, email evidence from TC-06 satisfies AC-1 context for this env.

**Steps**

First open **Logs** in **`stage-insights`**.

Then query traces/exceptions for **Wellfit Treasury API** in the time window of TC-05, filtering by **`correlationId`** when possible.

Next confirm SFTP/connect failure telemetry (exception or error trace) is present.

Finally save query screenshot; note if structured batch fields are missing vs email body content.
