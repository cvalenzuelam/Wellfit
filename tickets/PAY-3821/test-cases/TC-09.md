Validate that Treasury API remains healthy after funding-batch send failure

**AC Linked:** PAY-3821 / AC-3

**AC Description:** Background service stays alive; health and send endpoint remain usable after handled transport failure.

**Preconditions**

- TC-05 through TC-07 completed (failure path executed).

**Expected**

- **GET /health** returns **200** + **`Ok`** after the failure.
- **POST /send-funding-batch** still returns **202** (service not dead/host-loop crashed).

**Steps**

First send **GET** `/health` after the TC-05 failure run.

Then confirm **200** and body **`Ok`**.

Next send **POST** `/send-funding-batch` again with **`api_key`**.

Finally confirm **202** response (service still accepting work).
