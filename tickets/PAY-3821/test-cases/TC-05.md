Validate that send-funding-batch enqueues Send stage work under broken SFTP configuration

**AC Linked:** PAY-3821 / AC-1, AC-2 (failure trigger)

**AC Description:** Trigger returns 202 and correlation id while async send fails against dead SFTP host.

**Preconditions**

- TC-03 rows staged; TC-04 config applied.
- TC-02 auth pattern (`api_key`) configured in Postman.

**Expected**

- **POST /send-funding-batch** returns **202** with **`correlationId`**.
- After ~30 seconds settle time, async processing attempts SFTP connect (outcome asserted in TC-06–TC-09).

**Steps**

First send **POST** `/send-funding-batch` with header **`api_key`**.

Then capture **`correlationId`** from the **202** response body.

Next wait approximately **30 seconds** for background coordinator + SFTP connect timeout.

Finally retain **`correlationId`** for App Insights queries in TC-08.
