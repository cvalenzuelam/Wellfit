Validate that Treasury Azure configuration can induce SFTP failure and route alerts to QA inbox

**AC Linked:** PAY-3821 / AC-1 (failure trigger + alert delivery preconditions)

**AC Description:** Misconfigured SFTP host and settlement recipients are applied so the next send run fails at connect and alerts reach a verifiable inbox.

**Preconditions**

- Azure Portal access to **`stage-wf-treasury-api`** → Environment variables.
- TC-03 staged rows present.
- Original **`processor.vantiv.batch:sftpurl`** value noted for teardown (**`batch.vantivprelive.com`** in STAGE).

**Expected**

- **`processor.vantiv.batch:sftpurl`** set to **`10.255.255.1`** and applied (app restart if required).
- **`notifications:settlement:recipients`** includes the QA test inbox before failure trigger.

**Steps**

First open **`stage-wf-treasury-api`** application settings in Azure Portal.

Then set **`processor.vantiv.batch:sftpurl`** to **`10.255.255.1`**.

Next add or confirm the QA inbox in **`notifications:settlement:recipients`**.

Then click **Apply** and wait for any required app restart (~1 min).

Finally record both config changes (key + values) for test run evidence.
