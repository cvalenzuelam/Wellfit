Validate that Treasury API accepts send-funding-batch with api_key authentication

**AC Linked:** PAY-3821 / setup (enqueue path before async failure handling)

**AC Description:** Authorized trigger enqueues the Send stage and returns a correlation id for downstream log/DB checks.

**Preconditions**

- TC-01 passed.
- Collection variable `apiKey` matches the **Treasury API** collection `api_key` value.
- It is a **banking day** in the env calendar (otherwise the background loop may no-op).

**Expected**

- `POST /send-funding-batch` returns **202 Accepted**.
- Response JSON includes **`correlationId`** and **`stage`** = **`Send`**.

**Steps**

First set header **`api_key`** from the collection `apiKey` variable.

Then send **POST** `/send-funding-batch` with an empty or default body per collection.

Next capture **`correlationId`** from the response for later App Insights queries.

Finally confirm status **202** (not 403/404/503 unless queue-full is documented for the env).
